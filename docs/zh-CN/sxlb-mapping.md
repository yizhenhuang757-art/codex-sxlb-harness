# SXLB 映射与设计逻辑

SXLB 受三省六部的治理逻辑启发，但并不是历史制度模拟器。这些名称提供了一套容易记忆的分工词汇；它的运行目标则是当代的：让知识工作更容易被检查、质疑、协调、验证和续接。

## 核心转换

| 历史模型 | SXLB 角色或机制 | 被保留的部分 | 不作的主张 |
| --- | --- | --- | --- |
| 中书省 | 规划与综合 | 行动之前先有明确方案 | 复刻帝制诏令起草 |
| 门下省 | 独立复核、异议与完成复核 | 方案和完成声明都可以被质疑 | 历史上的封驳权或朝廷品级 |
| 尚书省 | 派发、协调与重新分配 | 获准的工作有明确负责人和边界工作包 | 控制真实人员的官僚机构 |
| 六部 | 功能服务边界 | 专门责任被区分并可见 | 固定的历史职掌，或一一等同的主张 |
| 史官式留痕 | 案卷记录、事件簿、状态面板和有边界的文档路由 | 决策可带着证据被重新审视 | 帝制档案，或自动发布个人数据 |

转换首先是一条次序，而不是一个头衔：**提案 → 独立复核 → 可追责执行 → 验证 → 留痕**。当任务只需要较轻路径时，SXLB 会降低工件的重量；它不会抹去决定、执行和接受之间的区别。

## 太子的作用

太子是现代编排角色，并非三省模型的直接组成部分。它接收请求、立案和路由，保持可见状态的一致，并在结案时汇总。它不能成为第二个规划部门，也不能越过尚书省直接派发部门拆分。

这个角色解决了智能体工作中的实际问题：对话需要可辨认的入口和出口，而不只是中间的一串专门阶段。

## SXLB 中的三省

### 中书省：在不可逆行动前先有方案

中书省将已立案的请求转化为包含目标、证据、约束、非目标、备选路径、验收标准、预算和停止条件的方案。关键设计在于：不确定性成为可以复核的文本。方案可以修订；隐藏假设不被视为执行授权。

### 门下省：区分提出异议与接受成果

门下省在实质执行之前复核方案，也在工作回传之后复核完成声明。它可以批准、要求补证、退回方案修订，或在普通复核会掩盖重要分歧时召集有边界的会审。这样，复核成为真实的决策界面，而不是事后的客套。

### 尚书省：派发是契约，而非标签

尚书省将获准方案转化为有边界的工作。派发应识别负责人、可写范围、交付物、验证方法、回传格式和集成者。证据变化时，它可以召回或重新分配工作。只给某部门命名、却没有工作包，不构成派发。

## 六部作为现代服务边界

| SXLB 部门 | 现代责任 | 被审慎借用的历史启发 |
| --- | --- | --- |
| 吏部 | 技能治理、能力评估和框架维护 | 人员任用及其责任适配性 |
| 户部 | 数据工作与分析 | 资源、账目和证据的保管 |
| 礼部 | 文档、面向使用者的清晰表达，以及起居郎回写 | 正式沟通、学习与制度呈现 |
| 兵部 | 集成与自动化 | 协调调动与运行支持 |
| 刑部 | 测试、调试、验证和技术发现 | 标准、程序、责任与可复核的发现 |
| 工部 | 实施与重构 | 技术建造和物质性执行 |

这些是为智能体治理工具而设的服务边界，并非对历史部权的翻译。一项任务可能使用一个部门、多个部门，或根本不拆分；判断标准是这种区分是否减少歧义、提升复核质量。

## 为什么公共版以文本为先

SXLB 将可读记录视为人、工具和智能体之间的共同界面。状态面板、方案、复核裁决、派发包和验证记录，使后来的读者无需依赖隐藏的模型状态，也能重建决策。

以文本为先并不等于只能手工操作。脚本可以准备清单、渲染状态、验证边界和呈现检查，但它们不能静默创造权力，不能把未复核的外部能力转为已批准能力，也不能取代复核裁决。

## 可选的多智能体转换路径

原始灵感可以支持多智能体工作，但 SXLB 不把派发当作默认项。受控制的路径是：

```text
翰林院参考池
  → 吏部评估与任用复核
  → 尚书省派发包
  → 有边界的智能体或技能执行
  → 合并摘要与验证
  → 门下省复核
```

外部智能体、技能、提示词或模式首先是参考材料。它们在成为实际工作候选项之前，会就适配性、安全性、许可证、所需工具和失败模式接受评估。真实派发需要正面的协调判断、相互独立的分支、明确负责人和可整合的书面回传；否则 SXLB 会保留本地或串行处理。

## 边界与限制

- 用户仍然拥有目标、范围、审批和外部承诺的决定权。
- 一个角色名称绝不授予工具权限、私有数据访问权或修改外部系统的权限。
- 框架追求适度治理：后果越重的工作，结构越多；小而有边界的任务，结构越轻。
- 历史语言是一套设计词汇和提问来源，不是现代智能体系统复刻帝制机构的证据。
- 公共文档发布协议和空白模板，绝不发布维护者的私有案卷、记忆存储、凭据或运行历史。

这正是 SXLB 的预期创新：将历史制度推理转化为现代工作的实用、可检查控制层，同时不把隐喻误当作身份。
