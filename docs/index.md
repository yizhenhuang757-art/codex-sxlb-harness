# Govern work without losing momentum

SXLB is an independent Codex skill harness inspired by [cft0808/edict](https://github.com/cft0808/edict) and its [task dispatch architecture](https://github.com/cft0808/edict/blob/main/docs/task-dispatch-architecture.md). It does not vendor or fork Edict or OpenClaw code. [Why SXLB exists]({{ '/why-sxlb/' | relative_url }}) explains the relationship, the differences, and the design choices.

SXLB uses a text-first route for work that needs clear proposals, independent review, accountable execution, and durable records.

> **Start here:** [Read the comparison]({{ '/why-sxlb/' | relative_url }}), inspect the [skill directory]({{ '/skill-directory/' | relative_url }}), or [install SXLB]({{ '/quickstart/' | relative_url }}). The public package includes empty, portable templates rather than a copy of anyone's private operating history.

## What you can expect

- A visible chain from request to proposal, review, execution, verification, and closure.
- A status board and case record that make current state and accountability easy to inspect.
- Optional multi-agent or multi-skill work only when coordination is worth its cost.
